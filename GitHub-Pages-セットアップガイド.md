# GitHub Pages セットアップガイド

このファイルは、GitHubリポジトリの管理者がGitHub Pagesを有効化するための手順を説明します。

## 🚀 GitHub Pages を有効化する手順

### 1. リポジトリの設定ページを開く

1. GitHubで [NeVOgino/NeV-NeV-no-MADO](https://github.com/NeVOgino/NeV-NeV-no-MADO) を開く
2. 上部メニューの **「Settings」**（⚙️設定）タブをクリック

### 2. GitHub Pages を設定する

1. 左側のメニューから **「Pages」** をクリック
2. 「Source」セクションで以下を設定：
   - **Source**: `Deploy from a branch`
   - **Branch**: `main` （またはデフォルトブランチ）
   - **Folder**: `/ (root)`
3. **「Save」** ボタンをクリック

### 3. 公開URLを確認する

- 数分後、同じPages設定ページに公開URLが表示されます
- URL例: `https://NeVOgino.github.io/NeV-NeV-no-MADO/`

### 4. 動作確認

- 表示されたURLをブラウザで開いて、ウェブサイトが正しく表示されることを確認

## 📝 注意事項

- **公開/非公開**: GitHub Pagesはデフォルトで公開されます
  - プライベートリポジトリでも、GitHub Pagesは公開される可能性があります
  - 社内のみで使用する場合は、GitHub Enterprise やプライベートネットワーク内での運用を検討してください

- **更新の反映**: 
  - ファイルを編集してコミットすると、自動的に数分以内に反映されます
  - キャッシュの影響で即座に反映されない場合は、ブラウザで `Ctrl + F5` (Windows) または `Cmd + Shift + R` (Mac) を押してリロードしてください

## 🔒 セキュリティについて

社内情報を含むコンテンツの場合：

1. **リポジトリのアクセス制限**
   - Settings → Manage access で閲覧権限を設定

2. **GitHub Pagesの代替案**
   - 社内サーバーでホスティング
   - GitHub Enterprise の Private Pages 機能を使用
   - 認証が必要なホスティングサービスを使用

## 🆘 トラブルシューティング

### Q: GitHub Pages の設定が見つからない

**A**: リポジトリの管理者権限が必要です。権限を持つユーザーに設定を依頼してください。

### Q: ウェブサイトが表示されない

**A**: 以下を確認してください：
1. GitHub Pages が有効化されているか
2. 5〜10分待ってから再度アクセス
3. `index.html` がリポジトリのルートに存在するか
4. ブラウザのキャッシュをクリア

### Q: 404 エラーが表示される

**A**: 
1. URLが正しいか確認（末尾にスラッシュ `/` を付ける）
2. ブランチ設定が正しいか確認
3. `index.html` が正しい場所にあるか確認
