# NeVの窓 - 更新マニュアル（初心者向け）

## 📖 このガイドについて

このマニュアルは、プログラミング経験がない方でもGitHubを使って「NeVの窓」ウェブサイトを更新できるように作成されています。

---

## 🎯 できること

- ✅ お知らせの追加・編集
- ✅ リンクの追加・変更
- ✅ 日付の更新
- ✅ 文章の修正

---

## 🚀 更新の手順

### ステップ1: GitHubにログイン

1. ブラウザで [https://github.com](https://github.com) を開く
2. あなたのアカウントでログインする

### ステップ2: リポジトリを開く

1. [NeVOgino/NeV-NeV-no-MADO](https://github.com/NeVOgino/NeV-NeV-no-MADO) を開く
2. `index.html` ファイルをクリック

### ステップ3: ファイルを編集する

1. 右上の **鉛筆マーク（✏️）** をクリック
2. テキストエディタが開きます
3. 変更したい箇所を探して編集します

### ステップ4: 変更を保存する

1. ページ下部にスクロール
2. 「Commit changes」セクションで：
   - 上の欄：変更内容を簡単に説明（例：「2025年10月の人事異動を追加」）
   - 下の欄：詳細説明（オプション）
3. 「Commit directly to the main branch」を選択
4. 緑色の **「Commit changes」** ボタンをクリック

### ステップ5: 確認する

1. 数分待つ
2. ウェブサイトのURLを開いて変更が反映されているか確認

## 📝 よくある編集パターン

### パターン1: 新しいお知らせを追加する

**場所**: `index.html` の `<!-- 全員向けタブ -->` または `<!-- 職員向けタブ -->` セクション

**追加する内容**:
```html
<div class="info-item">
    <span class="date">2025.11.04</span>
    <div class="content">
        <h3>新しいお知らせのタイトル</h3>
        <p><a href="#" class="link">詳細資料へのリンク</a></p>
    </div>
</div>
```

**手順**:
1. 既存の `<div class="info-item">` ブロックをコピー
2. 一番上に貼り付け（新しいものが上に来るように）
3. 日付、タイトル、リンクを変更
4. 保存

### パターン2: リンクを変更する

**変更前**:
```html
<a href="#" class="link">マニュアル</a>
```

**変更後**:
```html
<a href="https://example.com/manual.pdf" class="link">マニュアル</a>
```

**手順**:
1. 変更したいリンクを検索（Ctrl+F または Cmd+F）
2. `href="#"` の部分を実際のURLに変更
3. 保存

### パターン3: 日付を更新する

**変更前**:
```html
<span class="date">2024.7.24</span>
```

**変更後**:
```html
<span class="date">2025.11.04</span>
```

**手順**:
1. 変更したい日付を検索
2. 新しい日付に変更
3. 保存

### パターン4: リストに項目を追加する

**追加する内容**:
```html
<li><a href="#" class="link">新しい資料名</a></li>
```

**手順**:
1. リスト（`<ul class="resource-list">` ～ `</ul>`）を探す
2. 既存の `<li>...</li>` をコピー
3. 適切な場所に貼り付け
4. 内容を変更
5. 保存

## ⚠️ 注意事項

### やってはいけないこと

❌ `<!DOCTYPE html>` や `<html>` などの最初の行を削除しない  
❌ `<div>` や `</div>` などのタグを削除しない  
❌ `"` や `>` などの記号を削除しない  
❌ インデント（行頭のスペース）を大きく変更しない

### 安全な編集のコツ

✅ 既存のコードをコピーして、中身だけ変更する  
✅ 少しずつ変更して、その都度保存・確認する  
✅ 間違えたら、GitHubの「History」から前のバージョンに戻せる  
✅ 大きな変更をする前に、ファイル全体をコピーして保存しておく

## 🔧 トラブルシューティング

### Q: 保存したのにウェブサイトが更新されない

**A**: 以下を確認してください：
1. 3〜5分待ってからページを再読み込み（Ctrl+F5 または Cmd+Shift+R）
2. GitHubの「Actions」タブでデプロイが完了しているか確認
3. ブラウザのキャッシュをクリア

### Q: 編集中にエラーが出た

**A**: 以下を試してください：
1. ページを再読み込み
2. もう一度編集画面を開く
3. それでもダメなら、管理者に連絡

### Q: 間違えて削除してしまった

**A**: GitHubで元に戻せます：
1. ファイルを開く
2. 「History」ボタンをクリック
3. 間違える前のバージョンを探す
4. 「View」をクリックして内容を確認
5. 右上の「...」メニューから「Revert this commit」を選択

### Q: HTMLタグがわからない

**A**: 基本的なタグ：
- `<div>...</div>`: グループ化
- `<h2>...</h2>`, `<h3>...</h3>`: 見出し
- `<p>...</p>`: 段落
- `<a href="URL">...</a>`: リンク
- `<ul>...</ul>`: リスト
- `<li>...</li>`: リストの項目

## 📚 もっと詳しく知りたい方へ

### GitHub Pages の設定

このウェブサイトはGitHub Pagesを使って公開されています。

**設定確認方法**:
1. リポジトリのトップページを開く
2. 「Settings」タブをクリック
3. 左メニューの「Pages」をクリック
4. 「Source」が「main」ブランチ、「/ (root)」に設定されているか確認

### ファイル構成

```
NeV-NeV-no-MADO/
├── index.html          # メインのHTMLファイル（これを編集）
├── style.css           # デザイン設定（通常は編集不要）
├── script.js           # 動作設定（通常は編集不要）
├── 更新マニュアル.md      # このマニュアル
└── README.md           # プロジェクト説明
```

### スタイルを変更したい場合

`style.css` を編集すると、色やフォントなどのデザインを変更できます：

**例：ヘッダーの色を変更**
```css
header {
    background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
    /* この行の色コードを変更 */
}
```
